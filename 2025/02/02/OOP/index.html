物件導向程式設計（OOP）

發表於 2025-02-02 分類於 學習紀錄

今天來複習一下在過去求學經歷中和工作中都有同學或同事覺得複雜的物件導向程式設計（Object-oriented Programming, OOP）。OOP 是程式設計典範 (Programming paradigms) 的其中一種，還有諸如函式程式設計（Functional Programming）、程序式程式設計（Procedural Programming）…等等。而值得注意的是，這些典範在使用上並非互斥，而是針對相同、或不同問題而產生出來的解決方法。同時，沒有絕對完美的設計典範，也不是何種問題只能依賴特定的典範來解決，面對一個問題的時候，只有相對適合的方法，就看當下我們或是專案架構統一想要採用哪種方式來解決問題。

本文會以Python作為範例語言進行實作。

什麼是 OOP？

核心概念就是「模擬真實世界的事物，並定義其屬性及操作方法」。物件（object）本身包含「資料」和「方法」，以現實世界來說「資料」就是資訊、特徵、屬性、定義，而「方法」就是該物件和其他事物的互動方式。

舉例有個物件是汽車，他的資料包含：

  • 廠牌：Toyota
  • 車型：INNOVA
  • 出廠年份：2018
  • 里程數：15000
  • 排氣量：1998cc
  • 油箱容量：55L
  • …

汽車的互動方法包含：

  • 行駛
  • 轉彎
  • 頭燈啟用
  • …

可以看到資料以「屬性」和「值」的方式對應出現。另外，「值」通常是有單位的，才能表示他的意義。例如油箱容量是公升而非毫升。而若今天沒有單位，可能就不知道是55公升還是55毫升。

開頭有提到，其實有多種不同的程式設計典範，可以處理各種問題。每個典範都有他的優缺與適用情境，然而「物件導向程式設計」還是被廣為使用和討論，因為它在面對現實世界複雜問題的時候，有比較好的維護性、擴展性和重複使用性。另外也是整理收納，隱藏程式細節，並且只提供清楚的應用方法，讓主程式變得簡潔，大幅提高可讀性。當然，他也是有缺點的，例如程式碼規模相對較大，需要投入較多的時間和心力去建構。

接著會介紹 OOP 常提到的

  • 概念：類別、介面
  • 特徵：封裝、抽象、繼承、多型

什麼是多型？

是指子類別繼承父類別的方法，但可能會有不一樣的處理方式。承上的例子，汽車和機車都可以行駛，但行駛的油耗並不相同。

為什麼要有類別？

類別能夠幫助我們快速建立相同類型的物件，節省了重複輸入的時間和心力，同時還有其他好處

  • 把「類似的事物」歸類成一類之後，未來能夠快速透過這個類別來建立這些物件
  • 如果已經有建立好的類別，那麼我們就可以直接使用，不需要自己建立物件、或重新建構一個類別
  • 如果要針對某一類的事物進行修改，那麼我們可以直接從類別下手修改，而不需要進入程式碼逐條修改

這就很像生物學當中的「界門綱目科屬種」的層層分類方式，透過這種方式讓我們可以用結構化的方法撰寫程式碼。這也是OOP之所以較易於維護的原因之一。然而，若在建構程式碼時定義的分類方法不明確，則也會影響維護的方便性。

抽象類別

奇怪的繼承

上個段落的最後我們提到了分類的好壞會影響我們維護的困難度，例如Motorcycle繼承Car的父類別，而若是未來新增了直升機、遊艇並且也都繼承了Car，雖然實作出來的結果或許可以如預期，但是看起來就非常的奇怪，而且沒有邏輯。

如果我們今天繼續充實Car類別，加入幾個門的屬性（四門汽車、雙門汽車），結果就會發現Motorcycle子類別也繼承了有幾個門的屬性，機車居然有門？！？！

子類別是父類別的其中一種

雖然我們可以任意分類、抽取屬性和方法建立類別。任意的繼承某個父類別來取得需要的屬性和方法，但這樣的「分類整理」毫無邏輯也稱不上結構化，最終只會造成各種混亂和錯誤。所以通常在建立父類別和繼承的時候，會遵循著「子類別是父類別的其中一種」規則，例如

  • 機車是載具的一種
  • 汽車是載具的一種
  • 止痛藥是藥品的一種
  • 感冒藥是藥品的一種
  • …

這樣一來可以比較有結構的模擬現實世界的各種事物，也比較不會發生錯誤。

抽象類別

在我們充實父類別時，想新增一個drive，但這時候發現，因為法規的限制，機車不能上國道，而汽車也不能駛入機車專用道。這樣該怎麼在父類別進行定義呢？在其他程式語言裡面會使用介面（Interface）一種強制規範的類別，然而 python 中沒有這個類別，因此我們通常使用abc來建立這個功能。

在drive方法上，我們用@abstractmethod裝飾器定義抽象方法（abstract method），主要目標是強制子類別必須實作這些方法，否則無法建立物件。

封裝（Encapsulation）

封裝（Encapsulation）是OOP的核心概念之一。主要的目標是將屬性和方法封裝在類別內，並透過存取控制來確保資料不會被外部隨意修改。以前面的範例來說，Vehicle的owner、license_plate是可以在任意地方被手動修改的。現實世界來看，這是很不合理的，車牌和車輛擁有者是必須在指定單位申請後才能作變更登記，並不是隨時想改就能改的。

這裡主要分享

  • 私有屬性(Private Attribute)
  • 私有方法(Private Method)

私有屬性(Private Attribute)

在類別中可以進行存取，而外部無法存取的屬性。使用方式就是在私有屬性前加上兩個底線(__)

從上面的範例可以看到，我們在類別外無法存取到類別中的owner_id私有屬性。但是Python不像其他程式語言真的把屬性或方法變為私有，事實上我們還是可以透過加上_類別名稱存取到私有屬性。

所以在類別中的屬性或方法加上兩個底線(__)主要是告知此屬性或方法為私有，避免直接或意外的存取。
然而真的有需要修改或查看私有屬性時，可以透過新增 getter 和 setter 來達成。

結語

透過類別與物件，我們可以構建結構清晰、易於維護的程式碼。就像整理衣服一樣，每個衣物都有它應該在的地方，你可以輕鬆地找到他們並搭配屬於自己的穿搭。
個人認為透過學習OOP可強化對程式碼之間關聯性的理解，也可以改善程式碼的品質與閱讀性。最後值得一再提及的是，邏輯合理且明確的分類是好的OOP開發的重要條件之一。一個清晰符合邏輯的定義可以大幅減少專案開發過程中的各種 error 和疏漏。
